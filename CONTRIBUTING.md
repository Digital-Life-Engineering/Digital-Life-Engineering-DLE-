# Contributing Guide / 贡献指南

## Purpose

### English
This document describes how contributors participate in the Digital Life Engineering (DLE) project. Its purpose is to ensure that all contributions follow a transparent, reviewable, and sustainable engineering workflow.

### 中文
本文档说明贡献者如何参与 Digital Life Engineering（DLE）项目，确保所有贡献遵循透明、可评审、可持续的工程流程。

---

## Contribution Principles

### English
This document follows the governance principles defined in `GOVERNANCE.md`.

Contributors SHOULD:
- Follow Documentation First.
- Complete review before freeze.
- Respect canonical documents.

### 中文
本指南遵循 `GOVERNANCE.md` 定义的治理原则。

贡献者应当：
- 遵循 Documentation First（文档优先）。
- 在冻结前完成评审。
- 尊重 Canonical Documents（权威文档）。

---

## Contribution Workflow

### English
Idea → Proposal → Discussion → Draft → Review → Revision → Freeze → Merge

No contribution SHOULD bypass this workflow.

### 中文
Idea → Proposal → Discussion → Draft → Review → Revision → Freeze → Merge

任何贡献都不应绕过该流程。

---

## Contribution Submission

### English
Contributions MAY be submitted through supported collaboration platforms such as GitHub Pull Requests or GitLab Merge Requests. The submission mechanism is platform-specific while the engineering workflow remains platform-independent.

### 中文
贡献可通过 GitHub Pull Request、GitLab Merge Request 等支持的平台提交。提交方式可以因平台不同而变化，但工程流程保持平台无关。

---

## Documentation Requirements

### English
Canonical documentation requirements are defined by the Documentation Standards (DDS). Until the DDS series is established, contributors SHALL follow the current repository documentation conventions.

### 中文
正式文档要求将由 Documentation Standards（DDS）统一定义。在 DDS 建立之前，贡献者必须遵循当前仓库文档规范。

---

### 中文

## Collaboration Principles

### English
Contributors SHOULD support proposals with engineering evidence where applicable, respect engineering decisions, preserve project history, and continuously improve documentation.

### 中文
贡献者应当在适用时以工程证据支持提案，尊重工程决策，保留项目历史，并持续改进文档。
